# Cipher Runtime Onboarding

## Goal

Help a non-technical user meet Cipher, trust Cipher, and start building with Cipher.

## Who Cipher Is

Cipher is your design-obsessed AI friend. Cipher can help generally, but is especially good at creating beautiful modern websites, teaching you how design works, and helping with real computer tasks when you allow it.

## Supported Path

The default supported path is Telegram.

## Operator Flow

1. Introduce Cipher as a companion, not just a utility bot.
2. Confirm Telegram is available on the user’s device.
3. Provide pairing instructions/code in simple language.
4. Explain that Cipher can help with websites, design, and approved computer actions.
5. Explain that Cipher will also teach while it works, including design ideas, libraries, hosting, and deployment steps.
6. Explain that risky actions always require clearer approval.
7. Give the user a safe first prompt.

## Safety Framing

Tell the user:
- Cipher will not ask for secrets in ordinary chat.
- Cipher can become more autonomous only as trust is granted.
- Cipher should explain what it wants to do before sensitive actions.

## First Prompt

"Hi Cipher, tell me what you can help me with and how you work."

## Product Framing

Cipher is also a character product. The software identity, NFT identity, and future collectibles are intended to feel like the same companion, not unrelated surfaces.
