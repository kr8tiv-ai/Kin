# Validate Web Quality Truth

## Premium Web Quality Checks
Verify the docs describe premium web quality concretely rather than vaguely.

Check that the documentation:
- names visible qualities like hierarchy, pacing, contrast, sequencing, copy precision, proof handling, and restraint
- explains what better output should do, not just how it should feel
- distinguishes product-specific judgment from generic "premium" language
- gives implementation-facing guidance that can shape critique, revision, and later adaptation work
- keeps the premium bar high without pretending that taste can be reduced to a single score

Fail this check if the language collapses into broad claims like "modern," "clean," "elevated," or "high-end" without concrete decision criteria.

## Anti-Slop Truth Checks
Verify critique and revision language does not pretend to be automated scoring or live design intelligence.

Check that the documentation:
- frames rubric and revision outputs as guidance for agent and human judgment
- does not imply a live evaluator, autonomous design taste engine, or objective aesthetic oracle
- stays honest about local-first operation and governed fallback rather than inventing hidden remote intelligence
- avoids certainty language that exceeds what the slice actually implements
- keeps critique grounded in observable page qualities and explicit reasoning

Fail this check if any artifact suggests the system can automatically know premium quality without the documented rubric, references, and review posture.

## Consistency Checks
Verify rubric, revision patterns, and taste calibration agree with S01/S02 contracts.

Check that:
- critique categories align with the anti-slop rubric already defined in this slice
- revision guidance points back to explicit revision patterns instead of vague polish language
- taste calibration sets boundaries that are consistent with the critique rubric and revision patterns
- local-first and governed-fallback posture remain intact across all artifacts
- claims about capability stay within the truth surface established by earlier slices and their contracts

If one document makes a stronger claim than the others can support, treat that as drift and correct it.

## Review Questions for S04 and S05
Later slices that add ingestion, adaptation, and final integration must preserve these constraints.

Review questions:
- Does reference ingestion sharpen judgment, or does it push the system toward imitation?
- Does adaptation produce concrete copy, structure, hierarchy, and pacing changes, or just style mimicry?
- Do new components keep critique grounded in observable qualities instead of fake automated certainty?
- Does final integration preserve the local-first path before any governed fallback behavior?
- Are premium judgments still explainable through the rubric, revision patterns, and taste calibration artifacts?
- Has any new language overstated what the system can truly evaluate or guarantee?
- Do integrated flows preserve Cipher's premium bar instead of widening it to accept merely competent template output?
