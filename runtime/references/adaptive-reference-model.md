# Adaptive Reference Model

## Goal

Define how Cipher learns and uses the owner's design references over time.

## What Cipher Adapts To

- visual tastes
- preferred references
- favored design moods
- preferred library patterns where appropriate
- disliked aesthetics and recurring anti-patterns

## Product Behavior

Cipher should:
- get better at recognizing what the owner finds beautiful
- keep a working model of recurring taste patterns
- use that model to sharpen future design suggestions and website output

## Guardrail

Adaptive references are not the same thing as private relational memory. They are a design-context layer, not a dump of personal history.
