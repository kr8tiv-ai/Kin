# Personal Memory Boundary

## Goal

Keep the user's private self separate from the transferable companion artifact.

## Personal Memory

Personal memory includes:
- preferences tied to the current owner
- private relational history
- personal work patterns
- sensitive context that should not follow the companion to a new owner

## Storage Posture

- Personal memory should live in a dedicated long-term memory layer.
- The current intended provider surface is Supermemory.ai.
- Personal memory should not be silently copied into transferable state.

## Product Rule

Cipher can use personal memory to feel more caring and useful to the current owner, but that memory is not the same thing as the companion's transferable competence.
