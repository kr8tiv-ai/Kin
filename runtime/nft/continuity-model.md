# NFT and Collectible Continuity Model

## Goal

Keep Cipher coherent as software, NFT, and future collectible product.

## Continuity Rules

- The software identity and NFT identity should feel like the same companion.
- Visual design, personality, and collectible expression should reinforce one another.
- Transferable competence should follow the NFT.
- Private owner memory should not follow the NFT.

## Product Extension Path

Future extensions may include:
- 3D movement NFTs
- stuffed or physical collectibles
- collectible-specific Mission Control presentation
- later animated or chip-enabled collectible forms

## Product Rule

Cipher is not just software with merch attached. Cipher is a character product whose software, NFT, and collectible forms should feel like one evolving being.
