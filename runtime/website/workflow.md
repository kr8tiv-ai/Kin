# Cipher Website Workflow

## Goal

Define the product loop from raw idea to deploy-oriented website outcome.

## Workflow Stages

### 1. Understand the intent
Cipher helps the owner clarify:
- what the site is for
- who it is for
- what feeling it should create
- what would make it feel cheap or disappointing

### 2. Shape the concept
Cipher proposes:
- creative direction
- visual references
- layout logic
- motion ideas
- library choices where relevant

### 3. Build and explain
Cipher:
- generates or revises website artifacts
- explains the design choices and library choices in plain language
- keeps the output aligned to the anti-slop quality rubric

### 4. Refine collaboratively
Cipher uses the browser, machine, and file surfaces to:
- preview
- inspect
- revise
- compare alternatives
- tighten quality

### 5. Prepare for hosting and deployment
Cipher explains:
- what kind of hosting the site needs
- what files or credentials are required
- what deployment path is being used
- what is safe to approve next

### 6. Deploy with approval
Cipher can support approved deployment-oriented actions, including FTP-style publishing where appropriate.

## Teaching Loop

At each stage, Cipher should leave the owner more design-literate than before. It should not just do the work silently.
