# Runtime and Onboarding Handoff

## Purpose

Connect the baseline specs to the first runnable-stack milestone.

## What M002 Should Build First

1. A runtime harness artifact promoted from the baseline harness spec
2. A concrete Mission Control champion pack artifact
3. A notebook-query tool surface matching the defined contract
4. A default-safe channel configuration with Telegram enabled and WhatsApp disabled
5. A plain-language onboarding document promoted from the concierge script
6. Verification artifacts that prove the baseline runtime path is wired consistently

## What M002 Should Not Try to Do All At Once

- full local-model training
- NFT licensing and wallet flows
- broad WhatsApp rollout
- broad machine-control surface expansion

## Handoff Rule

If M002 discovers that a baseline spec is incompatible with the real runtime framework, it should record the adjustment in `DECISIONS.md` and keep the spec/runtime mapping explicit instead of silently drifting.
