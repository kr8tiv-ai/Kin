# Specs

This directory holds baseline product and runtime-facing specification artifacts for KIN.

These files are design-time source material for implementation milestones. They define what should be built and how the major surfaces fit together, but they are not themselves the final runtime configuration unless later milestones promote them into executable assets.

## Files

- `kin-solo-harness.yaml` — baseline tenant/harness configuration draft
- `mission-control-pack.md` — governed prompt-pack and policy specification
- `notebook-query-contract.json` — NotebookLM query tool contract
- `safety-policy.md` — high-agency behavior and consent gating policy
- `onboarding-script.md` — concierge/non-technical onboarding script
- `local-model-strategy.md` — role of the specialized local creative-coding model
